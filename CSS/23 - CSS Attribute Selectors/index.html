CSS Attribute Selectors

Every major attribute selector pattern, demonstrated in one page.

1. [attr] — Attribute Presence

[disabled] styles any element that simply has the attribute, regardless of value.

2. [attr="value"] — Exact Value Match

input[type="text"] vs input[type="email"] vs input[type="password"]

3. [attr~="value"] — Contains Whitespace-Separated Word

[data-tags~="featured"] matches only if "featured" is one whole word in the list.

Card A: "new featured sale"
Card B: "clearance"
Card C: "best-featured" (should NOT match)

4. [attr|="value"] — Exact or Hyphen-Prefixed Match

[lang|="en"] matches "en" or "en-US", "en-GB", etc.

lang="en" → matched

lang="en-US" → matched

lang="en-GB" → matched

lang="bn" → NOT matched

lang="english" → NOT matched (no hyphen boundary)

5. [attr^="value"] — Starts With

a[href^="https://"] targets secure external links.

  • https://example.com
  • http://example.com
  • mailto:test@example.com
  • tel:+8801700000000

6. [attr$="value"] — Ends With

a[href$=".pdf"] and a[href$=".zip"] add file-type icons.

  • resume.pdf
  • archive.zip
  • photo.png
  • notes.docx

7. [attr*="value"] — Contains Substring Anywhere

a[href*="example.com"] matches the substring anywhere inside the value.

  • sub.example.com/page
  • another-example.com
  • unrelated-site.org

8. [attr="value" i] — Case-Insensitive Matching

[data-status="active" i] matches regardless of letter case.

ACTIVE
Active
active
inactive

9. Combining Multiple Attribute Selectors

input[type="text"][required] — both conditions must be true.
