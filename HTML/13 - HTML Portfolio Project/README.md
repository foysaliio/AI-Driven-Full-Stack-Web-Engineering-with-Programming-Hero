# HTML Portfolio Project

A personal portfolio website built using **pure HTML only** (no CSS/JS required) to
demonstrate a solid understanding of HTML's core concepts and semantic structure.

## 📂 File Structure

```
13 - HTML Portfolio Project/
│
├── assets/
│   ├── images/
│   │   └── profile.svg        # Placeholder profile photo (replace with your own)
│   ├── audio/
│   │   └── favorite-music.mp3 # Add your own audio file here
│   └── favicon/
│       └── favicon.svg        # Site favicon
│
├── index.html
└── README.md
```

## Features

- Semantic HTML
- Navigation
- Images
- Tables
- Forms
- Audio
- Video
- Iframe
- Meta Tags
- HTML Entities

## Sections Included

1. **Header** – Name and tagline
2. **Navigation** – Bookmark links to every section (`#about`, `#skills`, etc.)
3. **Hero Section** – Profile image (`figure` + `figcaption`) and short intro
4. **About Me** – Two short paragraphs about the author
5. **Skills** – Unordered list (technical skills) and ordered list (learning path)
6. **Education** – A table with Degree, Institute, Year, and Result
7. **Projects** – Five dummy projects, each with a placeholder GitHub link (`#`)
8. **Favorite Video** – Embedded YouTube video via `iframe`
9. **Favorite Music** – `audio` element with a placeholder source
10. **Contact Form** – Name, Email, Subject, Message, and Submit button
11. **Footer** – Copyright, Email, GitHub, and LinkedIn links

## Notes

- Replace `assets/images/profile.svg` with your own photo.
- Replace `assets/audio/favorite-music.mp3` with your own audio file (or leave the
  placeholder — the `audio` tag will simply show "no file" until one is added).
- All GitHub project links use `#` as placeholders — update them with your real
  repository URLs.
- No CSS was used intentionally, per the project requirements — the focus here is
  100% on correct, semantic HTML structure.
